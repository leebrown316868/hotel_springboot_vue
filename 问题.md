# 测试
正确的调试方法应该是
  1. 第一时间查看浏览器Console的所有错误信息
  2. 检查组件的基本导入是否正确
  3. 使用无痕模式或清除缓存避免缓存问题
  4. 不要过度修改代码，专注于实际问题
 教训
  - 简单问题简单解决，不要过度复杂化
  - 先看错误信息，再进行调试
  - 清除缓存是解决前端问题的有效方法
  - Firefox的调试输出比Chrome更清晰
# 登录
管理平台不要注册功能，注册在登陆后台的管理系统内
保存登录状态
前端请求的是 /auth/login，但后端配置的是 /api/auth/login，注意前后端一致
问题1：登录页面"闪烁"无法进入系统 - Vue Router 4导航守卫使用返回值而不是next()回调，导致导航异常。解决：使用返回值语法：`router.beforeEach((to) => { ... return { name: 'login' } })`，不返回任何值表示允许导航
问题2：/api/auth/me返回500空指针异常 - AuthController.getCurrentUser()没有处理userDetails为null的情况。解决：添加null检查返回401状态码。
问题3：登录返回401 Invalid email or password - 旧数据库密码编码问题。解决：删除data/hotel.db文件，重启后端重新初始化数据库。
问题4：登录页面输入错误密码时页面刷新 - api.ts响应拦截器在收到401错误时会执行window.location.href重定向，导致登录页面刷新。解决：只在非登录页面时才处理401错误并重定向，登录页面的401错误由Login.vue组件自己处理。

# 管理员模块
## 系统管理
问题1：Vue Router警告"The next() callback in navigation guards is deprecated" - 使用了旧的next()回调语法。解决：移除next参数，使用返回值：需要重定向时返回路由对象，允许导航时不返回任何值。

问题2：async-validator警告"confirmPassword is required" - ChangePasswordDialog中confirmPassword不在form对象中，导致验证规则无法正常工作。解决：将confirmPassword作为临时字段加入form对象，提交前解构排除该字段。

## 订单与财务监管
仪表板页面图表容器显示空白 - ECharts在DOM未完全渲染时初始化导致容器尺寸为0，解决方法是先加载数据设置loading=false，然后用双重nextTick()等待DOM完全渲染后再初始化图表，并在图表容器使用内联样式style="width:100%;height:280px"确保明确尺寸。


## 前台员工模块
### 客房管理
  1. 路径不一致 - 前端API和后端Controller缺少 /api 前缀，Vite别名配置错误导致请求失败

  2. DTO字段缺失 - RoomRequest 缺少 capacity 字段导致房间创建失败

  3. 数据库schema过期 - 删除数据库后重建解决User实体新字段问题

  4. 房型配置不同步 - Rooms.vue使用硬编码的房型选项，与rooms-resource的动态配置不同步。解决：从系统设置API动态获取房型配置，所有页面使用统一数据源。

  5. 容量配置权限控制 - 容量只能由管理员在rooms-resource页面配置，Rooms.vue添加/编辑房间时容量自动从房型配置获取不可修改。解决：移除表单中的容量输入字段，添加容量提示文字。

  6. 编辑房间报错 - 点击编辑按钮报错"can't access property, roomTypesConfig.value is undefined"。解决：在handleEdit函数中添加房型配置加载检查，在模板中添加roomTypesConfig.value存在性检查。

  问题2：创建房间返回500错误 - 缺少capacity字段 前端RoomRequest和roomForm缺少capacity字段，而后端数据库该字段为NOT NULL约束。

  问题5：数据库表缺少capacity列，现有数据库rooms表没有capacity列，需要在应用启动时自动添加该列。

  问题6：房间列表未按房间号排序，后端RoomServiceImpl使用Sort.by("id")排序，改为Sort.by("number")即可按房间号排序。

  问题7：前端排序未生效，前端使用分页，且排序在客户端执行，修改后端排序逻辑才能正确显示。

  2. 更换房间状态 CORS 错误 - OPTIONS 预检请求没有返回 CORS 头，因为 Spring Security 的 CorsFilter没有在过滤器链中正确执行，需要使用 FilterRegistrationBean 设置最高优先级。

  3. 更换房间状态 CORS 错误 - 前端使用PATCH方法但后端CORS配置的allowed-methods中没有PATCH，导致OPTIONS预检请求被拒绝，解决方法是在application.yml中添加PATCH到allowed-methods列表（GET,POST,PUT,PATCH,DELETE,OPTIONS）。

# 登出功能
网页右上角铃铛按钮没功能和用户下拉箭头无法登出 - Layout组件中用户区域只有视觉效果（cursor-pointer），缺少点击事件和下拉菜单功能，解决方法是用el-dropdown组件替换静态用户区域，导入logout函数实现登出逻辑。

# Maven编译问题
Maven编译失败，annotationProcessorPath dependencies version为空 - maven-compiler-plugin的annotationProcessorPaths中Lombok依赖缺少<version>标签。解决：在<properties>添加<lombok.version>1.18.24</lombok.version>，并在Lombok path中添加<version>${lombok.version}</version>。同时修复RoomTypeAlreadyExistsException和RoomTypeInUseException重复构造函数问题，删除RoomService中不存在的RemoteRequest导入，在RoomRepository添加countByType方法。

# 房型配置同步问题
问题1：rooms-resource页面编辑房型配置成功但数据库未更新 - 前端读取房型从room_types表，但更新调用/api/settings/room-types更新system_settings表的JSON字段，两个数据源不同步。解决：修改前端使用roomType API的updateRoomType方法直接更新room_types表。

问题2：rooms页面房间信息与数据库不同步 - RoomMapper.toResponse()返回枚举的displayName（中文名称），前端期望枚举名称（如SINGLE）来查找房型配置。解决：修改RoomMapper使用.name()代替.getDisplayName()返回枚举名称。
